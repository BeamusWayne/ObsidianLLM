# 操作手册

本地 LLM 知识库 · Web UI · 支持 Ollama / Claude API / 自定义中转站

---

## 目录结构

```
ObsidianLLM/
├── 操作手册.md          ← 本文件
├── 系统架构.md          ← 技术架构文档
├── schema.md            ← Wiki 结构规范（LLM 行为准则，勿随意修改）
├── settings.json        ← 所有配置（可通过网页 Settings 修改）
├── start.sh             ← 一键启动脚本
├── raw/                 ← 原始资料（只进不改）
│   ├── articles/        ← 网页文章 (.md)
│   ├── papers/          ← 论文 (.pdf 或 .md)
│   └── images/          ← 图片（Clip 下载 + 独立截图）
├── wiki/                ← LLM 自动维护，不要手动编辑
│   ├── index.md
│   ├── log.md
│   ├── concepts/
│   ├── entities/
│   └── topics/
├── scripts/
│   ├── llm.py           ← LLM 调用层（链式 fallback）
│   ├── ingest.py        ← 原料 → wiki，含后台 Worker
│   ├── clip.py          ← URL 剪藏
│   └── query.py         ← 对 wiki 提问
├── server.py            ← FastAPI 后端
├── frontend/            ← Vue 3 前端
├── .processed.json      ← 已处理文件记录（自动维护，勿删）
├── .ignored.json        ← 用户标记忽略的文件（自动维护）
└── .ingest_status.json  ← 后台 Worker 状态（自动维护）
```

---

## 启动服务

```bash
cd /Users/katya/Files/ObsidianLLM
./start.sh
```

脚本会自动：
1. 编译前端（`frontend/` → `frontend/dist/`）
2. 启动 FastAPI 服务（端口 8000）
3. 后台 Ingest Worker 自动启动，断点续传未处理文件

打开浏览器访问：**http://localhost:8000**

关闭服务：终端里按 `Ctrl+C`

## 日常工作流

### 第一步：剪藏网页

在网页界面 **Clip & Ingest** 页面，粘贴 URL → 点 **Clip**：

- 自动抓取文章正文和图片，保存到 `raw/`
- 如果「自动 Ingest」开关已开启 → 立即自动入队后台处理
- 如果「自动 Ingest」关闭 → 只保存文件，等待手动选择处理

也可以手动放文件到 `raw/`（PDF / .md / .txt / 图片），服务器重启时自动检测。

---

### 第二步：管理 Raw Files

右侧 **Raw Files** 面板支持精细操作：

**文件状态（颜色区分）：**

| 状态 | 颜色标签 | 说明 |
|------|---------|------|
| 待处理 | 蓝色 | 等待 Ingest |
| 处理中 | 蓝色 + Spinner | 正在被 Worker 处理 |
| 已处理 | 绿色 | 已写入 wiki |
| 已忽略 | 灰色 | 用户标记跳过 |

**文件分组：**
- 文章（`raw/articles/`）：显示 📄 图标，展开可查看关联图片缩略图
- 图片（`raw/images/`）：显示缩略图预览

**批量操作：**
1. 勾选单个文件，或点「全选」跨组选择
2. 操作栏出现：
   - **处理 N** — 选中文件入队（已处理文件需确认重新处理）
   - **忽略 N** — 标记为已忽略，自动 Ingest 时跳过
   - **恢复 N** — 取消忽略，恢复为待处理

**排序：** 右上角下拉切换最新在前 / 最旧在前 / 文件名

**自动 Ingest 开关：** 面板顶部小 toggle，打开后立即将所有未处理文件入队

---

### 第三步：查看处理进度

**Clip & Ingest** 页面中间的 **Background Ingest** 卡片实时显示：

- 当前处理文件名
- 推理链（如 `Claude → Ollama`）
- 已用时 / 队列长度
- 实时 log（LLM 调用、切换 backend、写入 wiki 等）

关掉浏览器不影响处理——Worker 在服务器进程里运行。

---

### 第四步：浏览 Wiki

左侧边栏列出所有 Wiki 词条，点击查看。

**Graph 页面**：可视化词条关系图

- 支持拖拽、缩放（鼠标滚轮）、搜索节点
- 单击节点查看详情，双击直接跳转词条
- 左下角「⚙ 图形力学」面板可调整物理参数：
  - **节点间距** — 链接两端的目标距离
  - **斥力强度** — 节点间排斥力（负数越大越分散）
  - **连线强度** — 链接的弹性强度
  - **中心引力** — 图形向中心聚拢的力度
  - **节点大小** — 全局节点尺寸缩放

---

### 第五步：提问

```bash
# 基础提问
.venv/bin/python scripts/query.py "你的问题"

# 提问并把答案存为 wiki 词条
.venv/bin/python scripts/query.py "你的问题" --save
```

Query 只回答 wiki 里有的内容，没有则明确说明。先 ingest 相关资料再问。

## 设置

网页界面 **Settings** 页面可配置：

### LLM 后端（Clip & Ingest 页面模型选择器）

| 标签页 | 说明 |
|--------|------|
| 🖥 Ollama | 本地模型，全程数据不离本机，免费无限制 |
| ☁️ Claude | Anthropic Haiku/Sonnet/Opus，速度快，按量计费 |
| 自定义预设 | 已保存的自定义 API 配置 |
| ＋ 新建 | 填写 Base URL / API Key / 模型名称，保存为预设 |

自定义预设可**拖拽换序**。

---

### 推理顺序（Settings 页面）

系统按顺序尝试各后端，遇到余额不足自动切换到下一个：

```
默认：☁️ Claude API  →  🖥 Ollama
示例：🔗 自定义       →  🖥 Ollama  （完全不用 Claude）
示例：☁️ Claude API  →  🔗 自定义  →  🖥 Ollama
```

操作：拖拽 `⠿` 调整顺序，鼠标悬停点 `✕` 移除，底部按钮添加新 backend。

---

### Ingest 行为（Settings 页面）

| 选项 | 说明 |
|------|------|
| **Clip 后自动入队** | 开：Clip 后立即处理；关：只保存，手动选择处理 |

---

### 并行 Ingest（Settings 页面）

| 文件大小 | 默认路由 | 说明 |
|---------|---------|------|
| < 3000 字符 | Ollama | 短文章，本地跑够用 |
| ≥ 3000 字符 | Claude | 长文章，API 速度快 |

Workers 数量（默认 3）：同时处理的文件数，设为 1 则顺序处理。

## 自动降级说明

当链中某个 backend 余额不足时：
- **自动切换到链中下一个** backend 继续处理，不中断队列
- 页面顶部出现橙色横幅提示「Claude 余额不足，已自动切换到本地 Ollama」
- 充值后页面刷新，下次 Ingest 恢复使用 Claude
- 点横幅右侧 `✕` 可关闭提示

充值地址：https://console.anthropic.com → Plans & Billing

## 常见问题

**Q: 服务启动后页面打不开？**

确认端口 8000 未被占用：
```bash
lsof -i :8000
```

**Q: Ingest 出错显示「Background Ingest 出错」？**

查看页面 log 末尾错误信息。常见原因：
- Ollama 未启动 → `ollama serve`
- Claude 余额不足 → 已自动降级 Ollama
- 自定义 API 地址错误 → 检查 Settings 中的 Base URL

确认 Ollama 在运行：
```bash
curl -s http://localhost:11434/api/tags
```

**Q: 想重新处理所有文件（如修改了 schema）？**

```bash
.venv/bin/python scripts/ingest.py --all
```

**Q: 某个文件被标记忽略了怎么恢复？**

在 Raw Files 面板勾选该文件 → 点「恢复」按钮。

**Q: `.processed.json` 误删？**

```bash
.venv/bin/python scripts/ingest.py --all
```

记录自动重建，已有 wiki 词条只会被更新，不会清空。

## 注意事项

- **不要手动编辑 `wiki/`**：下次 ingest 可能覆盖修改
- **`schema.md` 改动影响全局**：修改后建议 `ingest --all` 统一重跑
- **`.processed.json` 勿删**：Worker 去重依据
- **`.ignored.json` 可编辑**：直接删除其中的路径可以恢复被忽略的文件
- **关闭浏览器不影响处理**：Worker 在服务器进程里运行，只要不关终端就会持续处理
